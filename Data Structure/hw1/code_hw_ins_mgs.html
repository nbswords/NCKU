1st assigment
Due 10/22 17:00

You are asked to write a program which can

  • read a natual number n, an integral lower bound a_min, an integral upper bound a_max, and a random seed rseed
  • pirntout a sequence of n random integer numbers with random seed rseed (i.e. A[0], A[1],..., A[n-1]) uniformly distributed in the range [a_min, a_max]
  • write a function duplicate_array(A,B,m,n) to copy the entire mxn array A to another mxn array B. (e.g. here in this assignment we assume an one-dimensional 1xn array, thus m=1. This function will be used again for some future assignments, so I suggest you to implement an mxn version rather than a 1xn version.
  • write a function printout_arrary(A,m,n) to printout the entire mxn array A (m=1 in this assignment. however, this function will be used again for some future assignments, so I suggest you to implement an mxn version rather than a 1xn version.)
  • write a function ins_sort(A,n) which sorts these n integers in DESCENDING order by insertion sort
  • printout the sorted array A by printout_array(A,1,n). (!Note!, DO NOT invoke printout_array(A,1,n) inside function ins_sort(A,n). You HAVE TO invoke printout_array(A,1,n) in the main function)
  • write a function mg_sort(B,1,n) which sorts these n integers in DESCENDING order by merge sort
  • sort these n integers by the mergesort, printout the sorted sequence

Screen Input: n, a_min, a_max, rseed
Screen Output:

first output the sorted sequence by insertion sort in one line, then output the sorted sequence by merge sort in one line

example: suppose your code is named as sort1, then when you execute sort1, you should see:

Please input [n, a_min, a_max, rseed]: (then you input, say, 5 10 100, 2, press enter)

then you should see 5 numbers randomly generated from the range [10,100] on the screen, say,

11 22 55 18 53

then you should see the sorted results which look like

By insertion sort:
55 53 22 18 11
By merge sort:
55 53 22 18 11

About random number generator:

  1. You may use whatever random number generator function you like (some are built-in functions by the language already). Whatever you use, you must make sure it compile/run in unix environment.
  2. Note that most of the random number function only produce a real random number between 0 and 1, so you have to try to convert the number to be in the range [a_min, a_max] by yourself
  3. For those who use C, mt110147.c contains several functions which gives random real numbers between 0 and 1. For those who use C++, rn.cpp also produces random real numbers between 0 and 1. You may copy them with some modification by yourself to produce random integer numbers between the given range [a_min, a_max].
  4. Note that a random seed is used to generate random numbers in a generator. Same random seed will give you same sequences of random numbers. If you want to produce different random number everytime you call the function, you need to specify another random seed as an input. The TA recitation hour will explain more details. I
